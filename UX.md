# VibCheck - UX 설계

## 디자인 원칙

비개발자 바이브 코더는 기술 용어에 위축되고, "내가 할 수 있는 건가?"라는 불안을 안고 있다.
복잡한 인터페이스보다 대화형이 자연스럽고, 결과를 빠르게 직관적으로 이해하고 싶어한다.

---

## 전체 플로우

```
[1] 랜딩/입력 → [2] 분석 중(스켈레톤+프로그레스) → [3] 리포트(벤토 그리드) → [4] 인라인 채팅
```

---

## [1] 랜딩/입력 — 원스텝 진입

히어로 영역에 단일 텍스트 입력 필드. 구글 검색처럼 단순하게.

- 헤드 카피: "만들고 싶은 게 있나요?"
- 입력 필드 하단에 예시 아이디어 칩 배치 (클릭 시 자동 입력)
  - "인스타 사진 분석해서 해시태그 추천하는 앱"
  - "유튜브 영상 자동 요약 크롬 확장"
  - "동네 맛집 리뷰 모아주는 서비스"
- CTA 버튼: "검증해보기"
- 서비스 설명은 스크롤 하단에 3단계로 간결하게:
  1. 아이디어를 알려주세요
  2. GitHub, 웹, AI가 분석합니다
  3. "만들 수 있어?"에 대한 답을 드립니다
- 이전 검증 히스토리 목록 (아이디어 제목 + 판정 등급 뱃지)

**핵심**: 첫 화면에서 바로 행동할 수 있어야 함. 설명을 읽어야 시작할 수 있는 구조는 피한다.

**기각된 대안**: 멀티스텝 가이드 입력 (플랫폼, 대상 등 추가 질문) → 진입 마찰 증가. 채팅에서 동일 정보 획득 가능.

---

## [2] 분석 중 — 스켈레톤 + 스텝 기반 프로그레스

사용자가 기다리는 10~20초가 이탈의 핵심 구간. 기다림을 경험으로 바꾼다.

### 스텝 프로그레스 (좌측 또는 상단)

```
 아이디어 분석 중...
  → "인스타그램 해시태그 추천 앱"으로 이해했어요

 GitHub에서 유사 프로젝트 찾는 중...
  → 관련 저장소 12개 발견

● 웹에서 시장 정보 수집 중...
  → Tavily 검색 진행 중

○ AI 리포트 생성 대기 중...
```

### 스켈레톤 프리뷰 (메인 영역)

분석 시작과 동시에 리포트의 벤토 그리드 뼈대를 미리 표시한다.
데이터가 도착하면 해당 타일부터 실제 콘텐츠로 교체된다.

```
분석 시작 직후:
┌──────────────┬──────┐
│ ░░░░░░░░░░░  │░░░░░ │  ← 펄스 애니메이션
│ ░░░░░░░░░░░  │░░░░░ │
├──────┬───────┴──────┤
│░░░░░ │ ░░░░░░░░░░░ │
│░░░░░ │ ░░░░░░░░░░░ │
├──────┴──────────────┤
│ ░░░░░░░░░░░░░░░░░░ │
└─────────────────────┘

데이터 도착 시 타일 단위 교체:
┌──────────────┬──────┐
│ 바이브코딩으로 │시장   │  ← 초록 fade-in
│ 가능!         │░░░░░ │
├──────┬───────┴──────┤
│░░░░░ │ ░░░░░░░░░░░ │  ← 아직 로딩 중
│░░░░░ │ ░░░░░░░░░░░ │
├──────┴──────────────┤
│ ░░░░░░░░░░░░░░░░░░ │
└─────────────────────┘
```

- 스텝 프로그레스와 스켈레톤을 동시에 표시
- "곧 여기에 결과가 채워진다"는 예측 가능성 → 이탈 방지
- 중간 결과("관련 저장소 12개 발견")를 미리 노출하여 기대감 형성

## [3] 리포트 — 벤토 그리드 + 판정 배지 퍼스트

### 레이아웃

데스크탑에서는 벤토 그리드로 정보 밀도와 시각 위계를 동시에 확보한다.
모바일에서는 자동으로 1열 스택으로 폴백.

```
데스크탑 (벤토 그리드):
┌──────────────────┬──────────┐
│                  │          │
│  [판정 배지]      │  시장     │
│  "바이브코딩으로   │  한 줄    │
│   가능!"          │          │
│  + 한 줄 요약     │          │
│                  │          │
├────────┬─────────┴──────────┤
│        │                    │
│ 필요한  │ Build / Fork /     │
│ 것들    │ Contribute         │
│        │                    │
│ - API  │ similar-repo 2.3k  │
│ - 도구  │ another-one 890    │
│        │                    │
├────────┴────────────────────┤
│                             │
│  구현 로드맵 (풀와이드 타임라인) │
│  Step 1 ──→ Step 2 ──→ Step 3│
│                             │
├─────────────────────────────┤
│ [공유] [새 아이디어]           │
├─────────────────────────────┤
│ 채팅 입력                    │
│ "궁금한 점을 물어보세요"       │
└─────────────────────────────┘

모바일 (1열 스택):
┌─────────────┐
│  판정 배지    │
├─────────────┤
│  한 줄 요약   │
├─────────────┤
│  필요한 것들  │
├─────────────┤
│  로드맵      │
├─────────────┤
│  B/F/C      │
├─────────────┤
│  시장 한 줄   │
├─────────────┤
│  채팅 입력    │
└─────────────┘
```

### 비주얼 스타일 — 글래스모피즘

카드에 반투명 배경 + backdrop-blur를 적용하여 깊이감을 준다.

- 리포트 카드: `background: rgba(255,255,255,0.05)` + `backdrop-filter: blur(12px)`
- 판정 배지: 등급 컬러에 맞는 은은한 글로우 이펙트 (초록 판정 → 초록 글로우)
- 헤더/네비게이션: 블러 배경으로 스크롤 시 콘텐츠와 자연스럽게 분리
- 저사양 기기 fallback: 블러 미지원 시 불투명 배경으로 대체

### 리포트 UX 상세

1. **판정 배지가 가장 먼저** — 스크롤 없이 답부터 본다. 색상으로 즉시 인지
2. **스트리밍은 타일 단위** — 벤토 그리드의 각 타일이 순차적으로 스켈레톤에서 콘텐츠로 교체
3. **유사 프로젝트는 카드형** — GitHub 링크, 스타 수, 활성도 한눈에. 새 탭으로 열림
4. **로드맵은 풀와이드 타임라인** — 수평 스텝퍼 (데스크탑) / 수직 스텝퍼 (모바일), 각 단계 클릭 시 상세 토글
5. **기술 용어에 툴팁** — "API" → "다른 서비스의 기능을 빌려 쓰는 방법이에요"

### 판정 배지 Reveal 모션

판정 등급이 공개되는 순간은 전체 경험의 클라이맥스다. 드라마틱하게 연출한다.

- 스켈레톤 상태에서 배지 타일의 컬러가 서서히 채워지는 연출
- 등급 텍스트가 페이드인되며, 배경에 해당 등급 컬러의 글로우가 퍼짐
- 한 줄 요약은 배지 등장 직후 타이핑 애니메이션으로 표시

## [4] 채팅 — 인라인 + 액션형 추천

리포트 하단에서 자연스럽게 채팅으로 이어지는 구조.

### 기본 구조

- 리포트 맨 아래에 채팅 입력 필드
- 질문하면 리포트 아래에 대화가 이어짐
- 스크롤 위로 올리면 리포트, 아래로 내리면 대화
- 리포트 각 섹션에 "이 부분 더 알아보기" 버튼 → 클릭 시 관련 질문 자동 생성

### 액션형 추천 칩

단순 질문("더 알려줘")이 아닌 **다음 행동을 제안**하는 칩을 제공한다.
바이브 코더는 "알고 싶은 사람"이 아니라 "만들고 싶은 사람"이다.

```
질문형 (기존):
├ "로드맵 2단계 자세히"
├ "더 쉬운 방법은?"
└ "범위를 줄이면?"

액션형 (추가):
├ "이 아이디어를 더 쉬운 버전으로 줄여줘"
├ "fork 추천 프로젝트의 시작 가이드 만들어줘"
├ "이 기술 스택으로 시작 프롬프트 작성해줘"
└ "로드맵을 체크리스트로 정리해줘"
```

- 맥락에 따라 질문형 + 액션형을 혼합 제공
- 리포트 내용 기반으로 동적 생성

### Markdown 렌더링

AI 응답은 Markdown으로 반환되며, 채팅 영역에서 리치 텍스트로 렌더링한다.

지원 요소:
- 헤딩 (`##`, `###`)
- 볼드/이탤릭/인라인 코드
- 코드 블록 (구문 강조 포함)
- 순서 있는/없는 리스트
- 테이블
- 링크 (새 탭으로 열림)

### 채팅 모션

- AI 응답 시 타이핑 인디케이터 (dot pulse)
- 메시지 등장: 부드러운 slide-up
- 추천 칩 등장: 응답 완료 후 순차 fade-in

**기각된 대안**: 사이드 패널 채팅 → 모바일에서 공간 부족, 맥락 분리됨.

## 모션 디자인 원칙

모션은 장식이 아닌 **커뮤니케이션 도구**다. 모든 모션은 "지금 무슨 일이 일어났는지"를 설명해야 한다.

| 순간 | 모션 | 목적 |
|------|------|------|
| 분석 시작 | 입력 필드가 상단으로 이동 + 스켈레톤 전개 | 상태 전환 인지 |
| 스텝 완료 | 체크 아이콘 애니메이션 + 중간 결과 slide-in | 진행 신뢰감 |
| 판정 공개 | 컬러 필 + 글로우 + 텍스트 fade-in | 클라이맥스 감성 |
| 타일 로딩 완료 | 스켈레톤 → 콘텐츠 crossfade | 자연스러운 전환 |
| 로드맵 스텝 간 | 연결선이 그려지는 애니메이션 | 순서와 흐름 전달 |
| 채팅 메시지 | slide-up + 타이핑 인디케이터 | 대화 리듬감 |
| 공유 클릭 | 체크 마크 + "복사됨" 피드백 | 액션 완료 확인 |

**금지**: 의미 없는 장식 모션, 로딩과 무관한 반복 애니메이션, 과도한 바운스/스프링.

## 부가 기능 UX

### 새 아이디어 검증
- 헤더에 "새 아이디어" 버튼 상시 노출
- 현재 맥락 이탈 확인 후 랜딩 입력 화면으로 이동

### 리포트 공유
- 리포트 상단에 공유 버튼 → 고유 URL 복사
- 공유 링크로 접속 시 리포트 읽기 전용
- 공유 받은 사람도 같은 아이디어로 새 검증 시작 가능

### 검증 히스토리
- 로컬스토리지에 이전 검증 목록 저장 (로그인 불필요)
- 랜딩 페이지 하단에 아이디어 제목 + 판정 등급 뱃지로 표시
- 클릭 시 해당 리포트 다시 열기

### 다크/라이트 모드
- 기본: 다크 모드
- 헤더에 토글 버튼
- 시스템 설정 자동 감지 지원

## 감성/톤 가이드라인

| 요소 | 방향 |
|------|------|
| 색상 | 다크 배경 기본 + 판정 등급별 악센트 컬러 |
| 표면 | 글래스모피즘 카드 (반투명 + backdrop-blur + 미세 보더) |
| 폰트 | Pretendard, 본문 16px 이상 |
| 톤 | 친구가 알려주는 느낌 ("~할 수 있어요", "~해보세요") |
| 아이콘 | 미니멀 라인 아이콘 + 섹션별 이모지 활용 |
| 모션 | 의미 있는 모션만 사용. fade/slide 기본, 판정 공개에서 강조 |
| 레이아웃 | 데스크탑: 벤토 그리드 / 모바일: 1열 스택 |
| 반응형 | 모바일 퍼스트, 데스크탑 확장 |
