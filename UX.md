# VibCheck - UX 설계

## 디자인 원칙

비개발자 바이브 코더는 기술 용어에 위축되고, "내가 할 수 있는 건가?"라는 불안을 안고 있다.
복잡한 인터페이스보다 대화형이 자연스럽고, 결과를 빠르게 직관적으로 이해하고 싶어한다.

---

## 전체 플로우

```
[1] 랜딩/입력 → [2] 분석 중(로딩) → [3] 리포트 → [4] 인라인 채팅
```

---

## [1] 랜딩/입력 — 원스텝 진입

히어로 영역에 단일 텍스트 입력 필드. 구글 검색처럼 단순하게.

- 헤드 카피: "만들고 싶은 게 있나요?"
- 입력 필드 하단에 예시 아이디어 칩 배치 (클릭 시 자동 입력)
  - "인스타 사진 분석해서 해시태그 추천하는 앱"
  - "유튜브 영상 자동 요약 크롬 확장"
  - "동네 맛집 리뷰 모아주는 서비스"
- CTA 버튼: "검증해보기"
- 서비스 설명은 스크롤 하단에 3단계로 간결하게:
  1. 아이디어를 알려주세요
  2. GitHub, 웹, AI가 분석합니다
  3. "만들 수 있어?"에 대한 답을 드립니다
- 이전 검증 히스토리 목록 (아이디어 제목 + 판정 등급 뱃지)

**핵심**: 첫 화면에서 바로 행동할 수 있어야 함. 설명을 읽어야 시작할 수 있는 구조는 피한다.

**기각된 대안**: 멀티스텝 가이드 입력 (플랫폼, 대상 등 추가 질문) → 진입 마찰 증가. 채팅에서 동일 정보 획득 가능.

---

## [2] 분석 중 — 스텝 기반 프로그레스

사용자가 기다리는 10~20초가 이탈의 핵심 구간. 기다림을 경험으로 바꾼다.

```
 아이디어 분석 중...
  → "인스타그램 해시태그 추천 앱"으로 이해했어요

 GitHub에서 유사 프로젝트 찾는 중...
  → 관련 저장소 12개 발견

● 웹에서 시장 정보 수집 중...
  → Tavily 검색 진행 중

○ AI 리포트 생성 대기 중...
```

- 프로그레스바 대신 스텝별 상태 표시
- 각 스텝 완료 시 체크 애니메이션 + 한 줄 중간 결과
- "관련 저장소 12개 발견" 같은 중간 결과를 미리 노출하여 기대감 형성
- "뒤에서 뭔가 열심히 일어나고 있다"는 신뢰감

---

## [3] 리포트 — 판정 배지 퍼스트

카드 기반 레이아웃, 위에서 아래로 순서대로 배치:

```
┌─────────────────────────────┐
│  [판정 배지]                  │
│  "바이브코딩으로 가능!"        │
│  신호등 컬러 (초록/노랑/주황/빨강) │
├─────────────────────────────┤
│  한 줄 요약                   │
│  "Clarifai API와 Next.js로   │
│   2주 안에 만들 수 있어요"     │
├─────────────────────────────┤
│  필요한 것들                  │
│  ├ Clarifai API (이미지 분석) │
│  ├ Next.js (웹 프레임워크)    │
│  └ Vercel (배포)             │
├─────────────────────────────┤
│  구현 로드맵                  │
│  Step 1 → Step 2 → Step 3   │
├─────────────────────────────┤
│  Build / Fork / Contribute   │
│  "이 프로젝트를 fork하면      │
│   더 빠를 수 있어요"          │
│  ├ similar-project  2.3k     │
│  └ another-one  890          │
├─────────────────────────────┤
│  시장 한 줄                   │
│  "관련 서비스 3개 존재,       │
│   수요 증가 추세"             │
├─────────────────────────────┤
│  [공유 버튼] [새 아이디어]     │
├─────────────────────────────┤
│  채팅 입력                    │
│  "궁금한 점을 물어보세요"      │
└─────────────────────────────┘
```

### 리포트 UX 상세

1. **판정 배지가 가장 먼저** — 스크롤 없이 답부터 본다. 색상으로 즉시 인지
2. **스트리밍은 섹션 단위** — 글자 하나하나가 아닌 카드가 순차적으로 fade-in
3. **유사 프로젝트는 카드형** — GitHub 링크, 스타 수, 활성도 한눈에. 새 탭으로 열림
4. **로드맵은 수직 타임라인** — 스텝퍼 UI, 각 단계 클릭 시 상세 토글
5. **기술 용어에 툴팁** — "API" → "다른 서비스의 기능을 빌려 쓰는 방법이에요"

---

## [4] 채팅 — 인라인 방식

리포트 하단에서 자연스럽게 채팅으로 이어지는 구조.

- 리포트 맨 아래에 채팅 입력 필드
- 질문하면 리포트 아래에 대화가 이어짐
- 스크롤 위로 올리면 리포트, 아래로 내리면 대화
- 리포트 각 섹션에 "이 부분 더 알아보기" 버튼 → 클릭 시 관련 질문 자동 생성
- 추천 질문 칩: "로드맵 2단계 자세히", "더 쉬운 방법은?", "범위를 줄이면?"

**기각된 대안**: 사이드 패널 채팅 → 모바일에서 공간 부족, 맥락 분리됨.

## 부가 기능 UX

### 새 아이디어 검증
- 헤더에 "새 아이디어" 버튼 상시 노출
- 현재 맥락 이탈 확인 후 랜딩 입력 화면으로 이동

### 리포트 공유
- 리포트 상단에 공유 버튼 → 고유 URL 복사
- 공유 링크로 접속 시 리포트 읽기 전용
- 공유 받은 사람도 같은 아이디어로 새 검증 시작 가능

### 검증 히스토리
- 로컬스토리지에 이전 검증 목록 저장 (로그인 불필요)
- 랜딩 페이지 하단에 아이디어 제목 + 판정 등급 뱃지로 표시
- 클릭 시 해당 리포트 다시 열기

### 다크/라이트 모드
- 기본: 다크 모드
- 헤더에 토글 버튼
- 시스템 설정 자동 감지 지원

## 감성/톤 가이드라인

| 요소 | 방향 |
|------|------|
| 색상 | 다크 배경 기본 + 판정 등급별 악센트 컬러 |
| 폰트 | Pretendard, 본문 16px 이상 |
| 톤 | 친구가 알려주는 느낌 ("~할 수 있어요", "~해보세요") |
| 아이콘 | 미니멀 라인 아이콘 + 섹션별 이모지 활용 |
| 애니메이션 | 기본 fade/slide, 로딩 단계에서만 강조 |
| 반응형 | 모바일 퍼스트, 데스크탑 확장 |
